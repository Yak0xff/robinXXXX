# 👨‍💻‍ 欲望越低，幸福感越高

 最近听到了一些关于自己所在业务组的不好消息，整个人不能说不担忧，但是难免有些焦虑，毕竟对于自己而言，年龄是现实，妥妥的中间人的尴尬，但是归根结底，自己也并不能左右什么，只能在自己的能力范围内，做好每一件事情，还是那句时刻保持自己的热情，其他就听天由命吧。

 面对幸福感，每个人的感受和想法都不尽相同，偶尔看到以上的芒格的一句：**能力 - 期望（欲望）** 是幸福的公式的时候，貌似明白了一点点。这里只记录下自己的一点点想法。

 欲望本身不可怕，可怕的是无穷无尽的欲望。人要是没了欲望，可能就没有生活的意义了吧。但是当无穷无尽的欲望占据了自己心灵的时候，生活会被压抑、紧张占据，那每天估计也是七零八碎的了。再说能力，能力这东西可谓千千万万中，有人说话的能力属实强大，各种场合各种情况，思维缜密，说话艺术，往往几句就能到处其中深意，扭转时局，这种能力常被人成为沟通或交际能力；也有人编程能力强悍，无论业务代码还是问题定位修复，往往一点即通，三两下即解决问题；还有等等能力，在能力上，可能大多数都认为是养家的能力，赚钱的能力等，这些能力大的时候，同时降低期望，所获得的幸福感往往更好更高。无论是家庭琐事，还是生活烦事，情绪稳定，有条理的去化解问题。

 当欲望大于能力，幸福感低不说，往往的到的结果就是无尽的焦虑。隐私控制焦虑的路径，要么降低欲望，同时不要有太高的期望，要么提升能力，让自己可以更好的做更多有利于实现期望的事情。还有一点往往会被忽略，就是要保持耐心，耐心靠着不断地坚持和坚定的毅力来维持，而这两种潜质需要靠不断地提升认知，培养眼光，具有长远眼光才能够判断一件事情的发展规律，拿捏本质，不迎头乱撞。

# ✍️ Git的基本工作方式

作为软件行业的从业者而言，代码的版本控制不能不知，在历史的进程里面，有SVN，但是由于其不适应不断发展的软件工程，最终被git所代替，目前git的时候非常流行，至于随着软件工程的发展，会有什么新的版本控制系统出现，目前不得而知了。关于git的使用，可以使用下面几张图来概括，自己也算是复习下吧。

![](/static/yk.2/git_1.png "Git 是如何工作的？") 

首先，必须确定代码的存储位置。常见的假设是只有两个位置 - 一个位于 Github 等远程服务器上，另一个位于我们的本地计算机上。然而，这并不完全准确。 Git 在我们的机器上维护了三个本地存储，这意味着我们的代码可以在四个地方找到：

- 工作目录：我们编辑文件的地方
- 暂存区：为下一次提交保留文件的临时位置
- 本地存储库：包含已提交的代码
- 远程存储库：存储代码的远程服务器

大多数 Git 命令主要在这四个位置之间移动文件。

<video class="video-container" src="/static/yk.2/git_2.mp4" controls="controls"></video>

Git的分支可以认为是你的每一项工作任务，可以完美的隔离开，互不影响。合理的使用分支将会事半功倍。

![](/static/yk.2/git_3.jpeg "Git 分支的删除") 

无用分支一定要记得及时删除，因为分支的数量上升是很巨大的，往往分支的数量不仅影响拉取的速度，还影响自动化流水线的效率等。
